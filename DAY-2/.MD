
## Step 1: Current Database Context

You are already inside MongoDB Shell and currently in a database context (example: `sampledb`).

---

## Step 2: Switch to `test` Database

Command used:

```bash
use test
```

Output:

```text
switched to db test
```

👉 MongoDB switched the active database to `test`.

---

## Step 3: Display All Databases

Command:

```bash
show dbs
```

Output:

```text
admin   40.00 KiB
config  96.00 KiB
local   40.00 KiB
```

### Explanation

* Only databases with **at least one collection** are displayed
* `admin`, `config`, and `local` are **default system databases**
* `test` is not shown because it has **no collections yet**

---

## Step 4: Switch to a New Database (`eShopping`)

Command:

```bash
use eShopping
```

Output:

```text
switched to db eShopping
```

👉 Since `eShopping` does not exist, MongoDB **creates it automatically** and switches to it.

---

## Step 5: Check Currently Selected Database

Command:

```bash
db
```

Output:

```text
eShopping
```

### Explanation

* `db` returns the name of the **currently selected database**
* Confirms that MongoDB is now operating on `eShopping`

---

## Step 6: Why `eShopping` Is Not Visible in `show dbs`

If you run:

```bash
show dbs
```

❌ `eShopping` will **NOT** appear yet.

### Reason:

* The database has **no collections**
* MongoDB shows a database **only after at least one collection is created**

---

## Step 7: Create a Collection in `eShopping`

Command:

```bash
db.users.insertOne({ name: "Ganesh" })
```

✔ This does the following:

* Creates the `users` collection
* Inserts one document
* Makes the `eShopping` database permanent

---

## Step 8: Verify Database Creation

Command:

```bash
show dbs
```

Now you will see:

```text
eShopping
```

👉 The database appears because it now contains a collection.

---

## Key Concepts (Important for Exams)

```text
Database → Collection → Document → JavaScript Object
```

* MongoDB Shell uses a **JavaScript-like environment**
* Databases and collections are created **automatically**
* MongoDB is schema-less

---

## Final Summary

* `use dbName` → switches or creates database
* `show dbs` → shows only non-empty databases
* `db` → shows current database
* `insertOne()` → creates collection + document

---



## Step 9: Check Collections in `eShopping` Database

Command:

```bash
show collections
```

### Explanation

* Displays all collections present in the current database
* Initially, there may be **no collections**

---

## Step 10: Insert a Document into `users` Collection

Command:

```bash
db.users.insertOne({ name: "john", age: 28, gender: "male" })
```

Output:

```text
acknowledged: true
insertedId: ObjectId("6721006a73b39dc91686b01e")
```

### Explanation

* If `users` collection does not exist, MongoDB **creates it automatically**
* One document is inserted successfully
* MongoDB generates a unique `_id` (ObjectId)

---

## Step 11: Verify Collections After Insert

Command:

```bash
show collections
```

Output:

```text
users
```

👉 Confirms that the `users` collection is created.

---

## Step 12: Fetch Documents from `users` Collection

Command:

```bash
db.users.find()
```

Output:

```json
{
  _id: ObjectId("6721006a73b39dc91686b01e"),
  name: "john",
  age: 28,
  gender: "male"
}
```

### Explanation

* `find()` returns all documents in the collection
* Output is shown in document (JSON-like) format

---

## Step 13: Create a Collection Explicitly (`products`)

Command:

```bash
db.createCollection("products")
```

Output:

```text
{ ok: 1 }
```

### Explanation

* Creates an empty collection named `products`
* `{ ok: 1 }` indicates success

---

## Step 14: Check Collections Again

Command:

```bash
show collections
```

Output:

```text
products
users
```

---

## Step 15: Create a Capped Collection (`logs`)

Command:

```bash
db.createCollection(
  "logs",
  { capped: true, size: 10000, max: 100, autoIndexId: true }
)
```

Output:

```text
{ ok: 1 }
```

### Explanation

* `capped: true` → fixed-size collection
* `size: 10000` → maximum size in bytes
* `max: 100` → maximum documents allowed
* `autoIndexId: true` → automatically indexes `_id`

## Key Takeaways (Very Important)

```text
Database → Collection → Document → JavaScript Object
```

* `insertOne()` can create collections automatically
* `_id` is auto-generated and unique
* `createCollection()` is optional
* `drop()` deletes a collection permanently
* MongoDB Shell uses a JavaScript-like syntax

## TL;DR (Quick Revision)

* `find()` → fetch documents
